Project Idea

Overview

This project will see the development of a smartphone application (iOS and Android) that will provide users with the tools to reduce their carbon footprint by reducing their average household food wastage.

A range of tools will be provided to users that will allow them to plan meals efficiently and create shopping lists based on the contents of their fridge, freezers, and pantry. By doing so users would only be purchasing what is required for those meals.

Educational resources concerning food waste will also be provided. These resources would include tips as well as links to organisations that offer more sustainable products and services for consumers. This application currently has a work in progress title of Fridge Friend.

Motivation

Globally, one-third of the world's food is wasted. This waste accounts for eight per cent of global greenhouse gas emissions. If food waste were a country, it would be the third-largest greenhouse gas emitter behind the US and China.

In Australia alone, food waste accounts for more than five per cent of the country's greenhouse gas emissions. Around 7.3 million tonnes of food is wasted annually. This waste is equal to about 300 kg per person or one in five bags of groceries. That is about $3,800 worth of groceries per household per year.

As reducing food waste is the third most effective way to address climate change, users can utilise the 'Fridge Friend' to efficiently track the life cycle of their food inventory.

Description

Fridge Friend is an application designed to assist users in managing their food waste ny making use of tools such as meal plannners and intuitive grocery lists. The concept of this application is that by utilising these tools, only ingredients required for those meals are purchased, resulting in reduced food waste. Additionally, the application will have easy to follow tips and tricks to allow adjustments to the users lifestlye that will make a change to landfill contribution as a result of food waste and raise environmental awareness that will hopefully, in turn, encourage consumer decisions in all aspects of life. To further aid users in making those decisions, the application will provide information on companies that source sustainable alternatives and products , e.g. Bulk food store, local markets and farms.

The application will feature the following functions:

  • Meal planner
  • Fridge, freezer and pantry inventory
  • Food waste calculator
  • Shopping list
  • Recipe datbase
  • Tips on how to make more environmentally friendly choices
  • Information on current environmental pressures
  • Links to sustainable stores/organisations
  • Reward system, e.g. badges for completing milestones/goals

The meal planner provides users with the ability to plan a number of meals ranging from weekly meal prep, mid-week meals to dinner parties. When using the meal planner, user will be able indicate that they have prepared a meal. This information will then be used to update some of the other features offered, as listed below.

The pantry, fridge, and freezer inventory tool will allow users to record their pantry, fridge and freezer contents accurately. The user will have the ability to either enter this information manually or utilise the built-in label scanner that uses the product's barcode to locate the item. The intuitiveness of this feature means that each time a meal is indicated as prepared, the quantity of ingredients used in the meal is then reflected in the inventory in real-time.

A food waste calculator will also be available. Any food that users are not able to utilise or compost can be input to the calculator and based on the type and amount of food, the calculator will return a result in the form of Emissions of CO2-e (carbon dioxide equivalent). Users will than be able to track, in real-time the impact of their own personal food waste.

The shopping list feature will automatically determine the ingredients required for a recipe. This would be done by utlising the inventory feature to determine if the user already has some of the ingredients to hand. To assist in keeping the inventory updated, users will be have the option to use a ‘commit’ function that will then move those items from the shopping list to the inventory, updating existing stock quantities.

The recipe function will allow users to search a database of existing recipes or create/upload their own recipes to the application.

The tips and tricks section will include an array of information from collated from current environmental statistics and data. The purpose of this feature is to provide users with information on the realities of the impact food waste can potentially have on the environment, motivating them to take that impact into account when making decisions. This section will also include information how to live a more sustainable lifestyle, providing tips such as buying in bulk, utilising alternative items like glass/silicone straws to eliminate its single-use counterparts or how to compost produce so that it does not end up in landfill.

Users will also be provided with links and information to stores and organisations that provide sustainable products. Thes include grocery stores, products, recycling centres or even food rescue organisations, for example, Foodbank, OzHarvest, bulk food stores, local markets and farms.

The application will also feature a rewards system in the form of reward badges. These badges would be issued to the user for completing tasks/reaching goals, acting as a positve motivator for the user to continue to use the application and make positive choices when in comes to food waste and sustainability.

Tools and Technologies

As this project will be heavily code dependant, PC’s will be the primary hardware needed.

In order for this application to be compatible across multiple platforms, including iOS and Android, mobile application building software would be. Apache Cordova or PhoneGap as it is now known, would be a great choice of software as it offers a range of features including the ability to target multiple platforms with one code and being open-source.

Skills Required

Overall, with the right skills and knowledge, this project is feasible. To build this application, several different skillsets will be required. When it comes to mobile applications, perhaps the most important aspect, besides the contect, would be the UI. This is the first thing that the user will see and so needs to not only be user-friendy but aestetically pleasing and so UI design skills would be advantageous.

An understanding of programming languages will also be necessary, especillay if using PhoneGap as apps built with this software are based predominately off of HTML, CSS and JS.

A basic understanding of backend computingmay also be necessary for efficient database management as well as security implementation.

Outcome

The most tangible measure for this application's success would be the reduction of food waste not only by households, but potentially companies as well.

For example, it would be great if instead of filling multiple large dumpsters with edible produce

that then ends up in landfill, major grocery stores and fast-food restaurants could instead donate

this to food rescue organisations such as Foodwise or OzHarverst.

This application's ultimate end goal is to make enough of an impact so companies are influenced by the environmentally driven choices that users are making so that said companies re-think their current food waste strategies.

On an individual level, I hope that the lifestyle change will help the average user to find a deeper appreciation for their environment and the food that they consume.
